Primeiro há de se definir a modelagem de seu source. O field que vai representar a imagem pode ser:

dataType="image": o campo referencia uma imagem armazenada em uma instância do serviço de imagens. O serviço de imagens que é o dono dessa imagem e não o conteúdo. Se o conteúdo for removido a imagem continua existindo. Se o serviço de imagens (ou a imagem de lá) for removida, a imagem vai deixar de existir, apesar do conteúdo continuar existindo.
dataType="file": o campo referencia um arquivo, cujo dono é o conteúdo. Se o conteúdo for removido o arquivo também será removido.
No caso de dataType="image", não há API pública disponível para sua criação.

No caso de dataType="file", você deve criar um FileConfig, persistí-lo usando ManagerFactory.getFileManager().add(...). Após chamado o add, fileConfig.getId() terá o identificador desse arquivo. Esse identificador deve ser armazenado como o valor desse campo de dataType="file" na sua tabela de dados de negócio. O tipo da coluna para esse campo no banco de dados é char(32).
